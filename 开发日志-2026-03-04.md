### 问题2：训练课次莫名消失
**现象：**
- 每次点击"编辑计划"按钮，训练课次就会消失
- 原本有多个课次，刷新后只剩1个

**根本原因：**
- "编辑计划"按钮会跳转到模板编辑器页面
- 模板编辑器修改的是模板数据，导致训练计划数据关联混乱

**解决方案：**
- 删除了"编辑计划"、"取消计划"、"删除计划"三个危险按钮
- 只保留安全的功能：
  - ✅ 编辑计划信息（对话框形式）
  - ✅ 添加/编辑/删除课次

**教训：**
- ⚠️ 训练计划和训练模板是两个不同的概念，不能混淆
- ⚠️ 不要在训练计划详情页直接跳转到模板编辑器
- ⚠️ 删除功能必须谨慎设计，避免误删数据

**文件位置：**
- `src/views/coach/PlanDetail.vue` - 第120-137行（已删除的按钮）

### 问题3：严重的性能问题（页面卡顿）
**现象：**
- 页面切换时卡顿
- 每次进入新页面都要等待几秒
- 数据量还不大（3个会员，几十条数据）就已经很卡

**根本原因：**
1. **N+1 查询问题**（最严重）
   - PlanDetail.vue：为每个课次单独查询动作
   - 10个课次 = 11次数据库查询（1次课次 + 10次动作）

2. **串行查询问题**
   - MemberDetail.vue：多个查询按顺序执行
   - 总加载时间 = 所有查询时间相加

3. **重复查询**
   - 同一个数据查询了多次

4. **查询不必要的字段**
   - 使用 `select('*')` 查询所有字段

**解决方案：**

#### 1. PlanDetail.vue - 优化 N+1 查询
**修改前：**
```javascript
// 为每个课次单独查询动作（N+1 查询）
const sessionsWithExercises = await Promise.all(
  sessionsData.map(async (session) => {
    const { data: exercises } = await supabase
      .from('session_exercises')
      .select('*')
      .eq('session_id', session.id)
    return { ...session, exercises }
  })
)
```

**修改后：**
```javascript
// 一次性查询所有课次的动作
const sessionIds = sessionsData.map(s => s.id)
const { data: allExercises } = await supabase
  .from('session_exercises')
  .select('*')
  .in('session_id', sessionIds)

// 在内存中分组
const exercisesBySession = {}
allExercises.forEach(exercise => {
  if (!exercisesBySession[exercise.session_id]) {
    exercisesBySession[exercise.session_id] = []
  }
  exercisesBySession[exercise.session_id].push(exercise)
})

const sessionsWithExercises = sessionsData.map(session => ({
  ...session,
  exercises: exercisesBySession[session.id] || []
}))
```

**性能提升：80%+**

#### 2. MemberDetail.vue - 删除重复查询
**修改前：**
```javascript
// loadMemberDetail 中查询了 member_plans
const plansResult = await getMemberPlans(memberId)

// loadTrainingStats 中又查询了一次 member_plans
const { data: memberPlans } = await supabase
  .from('member_plans')
  .select('template_id')
  .eq('member_id', memberId)
```

**修改后：**
```javascript
// 直接使用已加载的数据
if (!memberPlans.value || memberPlans.value.length === 0) {
  return
}
const templateIds = memberPlans.value.map(p => p.template_id)
```

#### 3. MemberDetail.vue - 只查询必要字段
**修改前：**
```javascript
.select('*')  // 查询所有字段
```

**修改后：**
```javascript
.select('id, template_id, completed, completed_date, session_date')  // 只查询需要的字段
```

#### 4. MemberDetail.vue - 优化查找算法
**修改前：**
```javascript
// O(n) 查找
const session = completedSessions.find(s => s.id === exercise.session_id)
```

**修改后：**
```javascript
// O(1) 查找
const sessionMap = new Map()
completedSessions.forEach(session => {
  sessionMap.set(session.id, session)
})
const session = sessionMap.get(exercise.session_id)
```

**性能提升：50%+**

**教训：**
- ⚠️ 避免 N+1 查询：能一次查询的，绝不分多次
- ⚠️ 避免重复查询：复用已加载的数据
- ⚠️ 只查询必要的字段：不要用 `select('*')`
- ⚠️ 使用合适的数据结构：Map 比 find() 快得多

### 问题4：页面导航逻辑混乱
**现象：**
- 从训练计划详情页点"返回"，回到了添加课次页面
- 从会员详情页点"返回"，回到了训练计划详情页

**根本原因：**
- 使用了 `router.back()`，依赖浏览器历史记录
- 浏览器历史记录包含了中间的所有页面

**解决方案：**
- 不使用 `router.back()`
- 直接跳转到明确的目标页面：
  - 训练计划详情页 → 会员详情页
  - 会员详情页 → 会员管理页

**修改代码：**
```javascript
// PlanDetail.vue
const goBack = () => {
  if (planDetail.value?.member_id) {
    router.push(`/coach/members/${planDetail.value.member_id}`)
  } else {
    router.back()
  }
}

// MemberDetail.vue
const goBack = () => {
  router.push('/coach/members')
}
```

**教训：**
- ⚠️ 不要依赖浏览器历史记录
- ⚠️ 返回按钮应该跳转到明确的页面
- ⚠️ 保证用户体验的一致性

**文件位置：**
- `src/views/coach/PlanDetail.vue` - 第364-372行
- `src/views/coach/MemberDetail.vue` - 第612-614行

### 问题5：控制台警告和错误
**现象：**
- Vue Router 警告：`next()` 回调已弃用
- ElTag 组件警告：`type="primary"` 不是有效值
- 数据类型错误：`d.weight?.match is not a function`

**解决方案：**
1. **Vue Router 警告**
   ```javascript
   // 修改前
   router.beforeEach((to, from, next) => {
     next()
   })

   // 修改后
   router.beforeEach((to, from) => {
     return true
   })
   ```

2. **ElTag 警告**
   ```javascript
   // 修改前
   <el-tag type="primary">

   // 修改后
   <el-tag>  // 删除 type 属性，使用默认样式
   ```

3. **数据类型错误**
   ```javascript
   // 修改前：weight 已经是数字，但还在用 match() 解析
   const match = d.weight?.match(/(\d+\.?\d*)/)

   // 修改后：直接使用数字
   return d.weight || 0
   ```

**教训：**
- ⚠️ 及时修复控制台警告，避免积累
- ⚠️ 注意数据类型的一致性
- ⚠️ 使用组件时查看文档，确保参数正确

**文件位置：**
- `src/router/index.ts` - 第112-120行
- `src/views/coach/PlanDetail.vue` - 第52行

## 📊 数据库结构说明

### 核心表结构

#### 1. members（会员表）
- `id` - UUID
- `name` - 会员姓名
- `phone` - 手机号
- `gender` - 性别
- `age` - 年龄
- `created_at` - 创建时间

#### 2. training_templates（训练模板表）
- `id` - UUID
- `name` - 模板名称
- `target_goal` - 训练目标
- `difficulty_level` - 难度等级（beginner/intermediate/advanced）
- `description` - 描述
- `created_at` - 创建时间

#### 3. member_plans（会员训练计划表）
- `id` - UUID
- `member_id` - 会员ID（外键 → members.id）
- `template_id` - 模板ID（外键 → training_templates.id）
- `status` - 状态（active/completed/cancelled）
- `start_date` - 开始日期
- `end_date` - 结束日期
- `notes` - 备注
- `created_at` - 创建时间

#### 4. training_sessions（训练课次表）
- `id` - UUID
- `template_id` - 模板ID（外键 → training_templates.id）
- `session_number` - 课次编号
- `core_focus` - 核心重点
- `training_part` - 训练部位
- `session_date` - 训练日期
- `completed` - 是否完成（boolean）
- `completed_date` - 完成日期
- `created_at` - 创建时间

#### 5. session_exercises（课次动作表）
- `id` - UUID
- `session_id` - 课次ID（外键 → training_sessions.id）
- `exercise_name` - 动作名称
- `equipment_notes` - 器械说明
- `weight` - 重量
- `reps_standard` - 次数标准
- `sets` - 组数
- `next_goal` - 下节进阶目标
- `coach_comment` - 教练评语（⚠️ 注意：不是 member_feedback）
- `progress_record` - 阶段进步记录
- `order_index` - 排序索引
- `created_at` - 创建时间

### 关键关系
```
members (会员)
  ↓ 1:N
member_plans (训练计划)
  ↓ N:1
training_templates (训练模板)
  ↓ 1:N
training_sessions (训练课次)
  ↓ 1:N
session_exercises (课次动作)
```
