# 训练计划深度复制功能 - 测试指南

> 更新日期：2026-03-06
> 功能：分配训练计划时为每个会员创建独立副本，互不影响

---

## 📋 功能说明

### 修改前的问题
- 会员修改自己的训练计划时，会影响模板库中的原始模板
- 其他使用同一模板的会员也会受到影响

### 修改后的行为
- 分配计划时，系统会为每个会员创建独立的训练计划副本
- 包括：模板信息、所有课次、所有动作
- 会员修改自己的计划不会影响模板库
- 会员之间的计划互不影响

### 测试4：数据库验证

**在 Supabase Dashboard 中执行：**

```sql
-- 1. 查看张三的专属计划
SELECT
  id,
  name,
  is_template,
  member_id,
  coach_id
FROM training_templates
WHERE is_template = false
  AND member_id = (SELECT id FROM members WHERE name = '张三')
ORDER BY created_at DESC;

-- 2. 查看模板库的原始模板
SELECT
  id,
  name,
  is_template,
  member_id
FROM training_templates
WHERE is_template = true
ORDER BY created_at DESC;

-- 3. 验证课次是否独立复制
SELECT
  ts.id,
  ts.template_id,
  ts.session_number,
  tt.name as template_name,
  tt.is_template
FROM training_sessions ts
JOIN training_templates tt ON ts.template_id = tt.id
WHERE tt.member_id = (SELECT id FROM members WHERE name = '张三')
ORDER BY ts.session_number;
```

**预期结果：**
- ✅ 张三有独立的 `training_template` 记录（`is_template=false`）
- ✅ 张三的模板有独立的 `training_sessions` 记录
- ✅ 张三的课次有独立的 `session_exercises` 记录
- ✅ 原始模板保持不变（`is_template=true`）

## 🔍 常见问题排查

### 问题1：分配后模板库被修改了
**原因：** 可能是旧数据（分配时间在功能更新之前）
**解决：** 重新分配一次计划，新分配的会使用独立副本

### 问题2：分配速度很慢
**原因：** 模板包含大量课次和动作
**解决：**
1. 检查数据库索引是否创建（运行 `添加性能优化索引.sql`）
2. 如果仍然慢，考虑优化模板大小

### 问题3：保存失败
**原因：** 可能是数据库权限问题
**解决：** 检查 Supabase RLS 策略是否允许创建专属计划
