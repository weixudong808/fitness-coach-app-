## 🚧 当前正在进行的任务

### 训练记录助手功能

**目标：** 通过对话方式实时记录会员训练数据，直接写入数据库。

**核心需求：**
1. 教练说"开启XXX训练记录"，进入记录模式
2. 教练实时输入训练动作数据（如：卧推 80kg 10次 3组）
3. AI解析并暂存到内存
4. 教练说"结束"，一次性写入数据库
5. 课次状态标记为"待训练"，教练确认后手动改为"已完成"

**关键设计决策：**
- 每个会员一个"日常训练记录"模板
- 课次编号自动递增
- 训练过程中不需要权限（暂存到内存）
- 结束时一次性写入，只需一次权限确认
- 支持取消、查看、删除上一条动作

## 📊 数据库结构

### 核心表

#### members（会员表）
```sql
- id: UUID (主键)
- name: TEXT (会员姓名)
- phone: TEXT (手机号)
- created_at: TIMESTAMP
```

#### training_templates（训练模板表）
```sql
- id: UUID (主键)
- name: TEXT (模板名称)
- description: TEXT (描述)
- target_goal: TEXT (训练目标)
- duration_weeks: INTEGER (持续周数)
- sessions_per_week: INTEGER (每周课次)
- is_exclusive: BOOLEAN (是否专属)
- exclusive_member_id: UUID (专属会员ID)
- created_at: TIMESTAMP
```

#### training_sessions（训练课次表）
```sql
- id: UUID (主键)
- template_id: UUID (外键 → training_templates)
- session_number: INTEGER (课次编号)
- core_focus: TEXT (核心重点)
- training_part: TEXT (训练部位)
- completed: BOOLEAN (是否完成)
- completed_date: DATE (完成日期)
- created_at: TIMESTAMP
```

#### session_exercises（课次动作表）
```sql
- id: UUID (主键)
- session_id: UUID (外键 → training_sessions)
- exercise_name: TEXT (动作名称)
- weight: TEXT (重量，如：80kg)
- reps_standard: TEXT (次数标准，如：10次)
- sets: INTEGER (组数)
- coach_comment: TEXT (教练评语)
- order_index: INTEGER (排序索引)
- created_at: TIMESTAMP
```

#### member_plans（会员训练计划表）
```sql
- id: UUID (主键)
- member_id: UUID (外键 → members)
- template_id: UUID (外键 → training_templates)
- status: TEXT (状态：active/completed/cancelled)
- start_date: DATE (开始日期)
- end_date: DATE (结束日期)
- created_at: TIMESTAMP
```

## 🔧 重要技术决策

### 1. 字段命名统一
**决策：** 使用 `coach_comment` 而不是 `member_feedback`
**原因：** 语义更准确，这是教练的评语，不是会员的反馈
**影响文件：**
- `AddSession.vue`
- `EditSession.vue`
- `PlanDetail.vue`
- `TemplateEditor.vue`

### 2. 性能优化
**问题：** N+1查询导致加载慢
**解决方案：** 一次性查询所有数据，在内存中分组
**示例：**
```javascript
// ❌ 错误：N+1查询
const sessions = await supabase.from('training_sessions').select('*')
for (const session of sessions) {
  const exercises = await supabase.from('session_exercises').select('*').eq('session_id', session.id)
}

// ✅ 正确：一次性查询
const sessions = await supabase.from('training_sessions').select('*, session_exercises(*)')
```

### 3. 训练记录模式
**决策：** 批量操作，结束时一次性写入
**原因：** 避免频繁的权限确认，提升用户体验
**实现：** 暂存到内存 → 结束时批量写入数据库
