**版本：** 1.0
**日期：** 2026-03-05
**状态：** 需求已确认，待实现

## 🎯 功能概述

通过对话方式实时记录会员训练数据，AI自动解析并写入数据库。

**核心价值：**
- ✅ 教练无需手动填写表单
- ✅ 自然语言输入，快速便捷
- ✅ 自动解析和格式化数据
- ✅ 最小化权限确认次数

---

## 📋 需求确认记录

### 问题1：会员识别
**方案：** A+B 结合
- 每次说"开启XXX训练记录"
- 记住当前会员，直到说"结束"

### 问题2：课次选择
**方案：** 每个会员一个"日常训练记录"模板
- 查询会员是否有"日常训练记录"模板
- 如果有，使用现有的
- 如果没有，自动创建一个
- 课次编号自动递增

### 问题3：动作顺序
**方案：** 按输入顺序自动编号
- 第1个动作：order_index = 1
- 第2个动作：order_index = 2
- 依此类推

### 问题4：数据保存方式
**方案：** 批量操作
- 训练过程中暂存到内存（不写数据库）
- 结束时一次性写入数据库
- 只需要一次权限确认
- 保存时标记为"待训练"（completed=false）
- 教练下课后自己去数据库确认
- 确认无误后，手动改为"已完成"（completed=true）

### 问题5：新记录的归属
**方案：** 每个会员一个"日常训练记录"模板
- 模板名称：`{会员名}-日常训练记录`
- 例如：`张三-日常训练记录`
- 所有临时训练都记录在这个模板下
- 课次编号自动递增

### 问题6：输入格式
**方案：** 灵活识别，自动补全
- 顺序：重量、次数、组数
- 可以不加单位（自动补全）
- 动作数据后的文字作为教练评语

## 🔧 功能详细设计

### 1. 开启训练记录

**触发词：**
- `开启 [会员名] 训练记录`
- `开始记录 [会员名]`
- `记录 [会员名] 训练`

**执行流程：**
```
1. 查询会员信息
   - 从 members 表查询会员
   - 如果找不到，提示错误并退出

2. 查询/创建"日常训练记录"模板
   - 查询是否存在名为"{会员名}-日常训练记录"的模板
   - 如果不存在，创建新模板：
     * name: "{会员名}-日常训练记录"
     * description: "日常训练记录"
     * is_exclusive: true
     * exclusive_member_id: 会员ID

3. 确定下一个课次编号
   - 查询该模板下所有课次
   - 找到最大的 session_number
   - 下一个课次编号 = 最大编号 + 1

4. 初始化记录状态
   - 当前会员ID
   - 当前模板ID
   - 下一个课次编号
   - 训练日期：当前日期
   - 训练时间：当前时间
   - 动作列表：空数组

5. 显示确认信息
   - 会员名称
   - 训练计划：{会员名}-日常训练记录
   - 课次编号：第X次课
   - 训练日期：YYYY-MM-DD

6. 进入记录模式
```

**示例：**
```
用户：开启张三训练记录

AI：✅ 已开启张三的训练记录
📋 会员：张三
📅 训练计划：张三-日常训练记录
🎯 课次：第5次课
📅 训练日期：2026-03-05
⏰ 开始时间：14:30

请告诉我训练动作（格式：动作名称 重量 次数 组数）
```

### 2. 记录训练动作

**输入格式（灵活识别）：**
```
[动作名称] [重量] [次数] [组数] [备注]
```

**支持的格式：**
- `卧推 80kg 10次 3组`
- `卧推 80 10 3`
- `卧推 80/10/3`
- `深蹲 100kg 8次 4组 感觉不错`
- `硬拉 120 6 3 下次加重`

**解析规则：**

1. **动作名称：** 第一个词
2. **重量：** 第二个数字（自动添加kg）
   - `80` → `80kg`
   - `80kg` → `80kg`
   - `80.5` → `80.5kg`
3. **次数：** 第三个数字（自动添加"次"）
   - `10` → `10次`
   - `10次` → `10次`
   - `8-10` → `8-10次`
4. **组数：** 第四个数字（纯数字）
   - `3` → `3`
   - `3组` → `3`
5. **备注：** 剩余的所有文字
   - 存储到 `coach_comment` 字段

**执行流程：**
```
1. 解析输入的动作数据
2. 验证数据格式
   - 如果解析失败，提示用户重新输入
3. 添加到内存中的动作列表
   - order_index = 当前动作数量 + 1
4. 返回确认信息
```

**示例：**
```
用户：卧推 80 10 3

AI：✅ 已记录：卧推 80kg×10 3组

用户：深蹲 100kg 8次 4组 今天状态不错

AI：✅ 已记录：深蹲 100kg×8 4组
💬 备注：今天状态不错
```

## 📊 数据库操作

### 创建"日常训练记录"模板
```javascript
const { data: template, error } = await supabase
  .from('training_templates')
  .insert({
    name: `${memberName}-日常训练记录`,
    description: '日常训练记录',
    target_goal: '日常训练',
    duration_weeks: 0,
    sessions_per_week: 0,
    is_exclusive: true,
    exclusive_member_id: memberId
  })
  .select()
  .single()
```

### 创建训练课次
```javascript
const { data: session, error } = await supabase
  .from('training_sessions')
  .insert({
    template_id: templateId,
    session_number: sessionNumber,
    core_focus: '日常训练',
    training_part: '',
    completed: false,  // 标记为"待训练"
    completed_date: null
  })
  .select()
  .single()
```

### 批量插入动作记录
```javascript
const exercises = [
  {
    session_id: sessionId,
    exercise_name: '卧推',
    weight: '80kg',
    reps_standard: '10次',
    sets: 3,
    coach_comment: '',
    order_index: 1
  },
  // ... 更多动作
]

const { error } = await supabase
  .from('session_exercises')
  .insert(exercises)
```
