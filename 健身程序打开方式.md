## 重要提示

### ⚠️ 保持终端窗口开着
- 启动服务器后，**不要关闭终端窗口**
- 如果关闭了终端，网页就会打不开
- 需要重新执行上面的步骤 1-4

### 🛑 停止服务器
如果需要停止服务器：
- 在终端窗口按 `Ctrl + C`

### 🔄 重新启动
如果网页打不开（显示"连接被拒绝"）：
1. 回到终端窗口
2. 按 `Ctrl + C` 停止旧的服务器
3. 重新执行 `npm run dev`

## 常见问题

### Q: 为什么网页打不开？
**A:** 开发服务器没有运行。请按照上面的步骤重新启动。

### Q: 端口被占用怎么办？
**A:** 如果看到 "Port 5173 is in use" 提示，说明上次的服务器还在运行。解决办法：
1. 在终端执行：`lsof -ti:5173 | xargs kill -9`
2. 然后重新执行 `npm run dev`

### Q: 登录失败显示 "Failed to fetch" 怎么办？
**A:** 这是 Supabase 连接问题。解决办法：
1. 检查 `.env` 文件中的 `VITE_SUPABASE_ANON_KEY` 是否完整（应该以 `eyJ` 开头）
2. 修改 `.env` 文件后，必须重启开发服务器（按 `Ctrl + C` 停止，然后重新 `npm run dev`）

### Q: 忘记密码了怎么办？
**A:** 使用上面提供的测试账号密码登录
