## 重要提示

### ⚠️ 必须使用 Node 20.x 版本
- 本项目需要 Node 20.x LTS 版本
- 如果使用其他版本（如 Node 25.x），会导致依赖安装失败
- 每次启动前务必执行 `nvm use 20`

### ⚠️ 保持终端窗口开着
- 启动服务器后，**不要关闭终端窗口**
- 如果关闭了终端，网页就会打不开
- 需要重新执行上面的步骤 1-5

### 🛑 停止服务器
如果需要停止服务器：
- 在终端窗口按 `Ctrl + C`

### 🔄 重新启动
如果网页打不开（显示"连接被拒绝"）：
1. 回到终端窗口
2. 按 `Ctrl + C` 停止旧的服务器
3. 确认 Node 版本：`nvm use 20`
4. 重新执行 `npm run dev`

## 常见问题

### Q: 为什么网页打不开？
**A:** 开发服务器没有运行。请按照上面的步骤重新启动。

### Q: 启动时提示 "vite: command not found" 怎么办？
**A:** Node 版本不正确或依赖未安装。解决办法：
1. 确认 Node 版本：`nvm use 20`
2. 如果还是不行，重新安装依赖：`pnpm install`
3. 然后重新启动：`npm run dev`

### Q: 端口被占用怎么办？
**A:** 如果看到 "Port 5173 is in use" 提示，说明上次的服务器还在运行。解决办法：
1. 在终端执行：`lsof -ti:5173 | xargs kill -9`
2. 然后重新执行 `npm run dev`

### Q: 登录失败显示 "Failed to fetch" 怎么办？
**A:** 这是 Supabase 连接问题。解决办法：
1. 检查 `.env` 文件中的 `VITE_SUPABASE_ANON_KEY` 是否完整（应该以 `eyJ` 开头）
2. 修改 `.env` 文件后，必须重启开发服务器（按 `Ctrl + C` 停止，然后重新 `npm run dev`）

### Q: 忘记密码了怎么办？
**A:** 使用上面提供的测试账号密码登录

### Q: 系统崩溃或出现严重问题怎么办？
**A:** 从存档节点恢复。解决办法：
1. 查看 `3.4号健身程序开发节点.md` 文档
2. 按照文档中的"如何从此节点恢复"步骤操作
3. 或者执行：`git reset --hard 6dc5005`
