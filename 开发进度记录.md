### 🔧 关键技术实现

#### 专属计划 vs 模板计划

| 特性 | 模板计划 | 专属计划 |
|------|---------|---------|
| `is_template` | true | false |
| `member_id` | null | 会员ID |
| 显示位置 | 模板库、分配计划页面 | 仅在会员的训练计划列表 |
| 创建方式 | 训练计划模板页面 | 会员详情页 → 创建专属计划 |
| 保存后跳转 | 模板列表 | 会员详情页 |
| 自动分配 | 否 | 是（自动创建 member_plans） |

#### 数据流程

**创建专属计划流程：**
```
会员详情页 → 点击"分配新计划" → 选择"创建专属计划"
→ TemplateEditor.vue (mode=exclusive, memberId=xxx)
→ 填写计划信息和课次动作
→ 保存时：
  1. 创建 training_templates (is_template=false, member_id=xxx)
  2. 创建 training_sessions 和 session_exercises
  3. 自动创建 member_plans 记录
→ 返回会员详情页
```

### ⚠️ 容易犯的错误和注意事项

#### 1. 数据库相关

**RLS（行级安全）策略问题：**
- ❌ 错误：创建表后忘记设置 RLS 策略
- ✅ 正确：创建表后立即设置 RLS 策略，并测试权限
- 💡 提示：如果遇到 403 Forbidden 错误，首先检查 RLS 策略

**PostgreSQL 语法问题：**
- ❌ 错误：使用 `CREATE POLICY IF NOT EXISTS`（PostgreSQL 不支持）
- ✅ 正确：先 `DROP POLICY IF EXISTS`，再 `CREATE POLICY`

**NOT NULL 约束问题：**
- ❌ 错误：保存数据时缺少必填字段（如 `coach_id`）
- ✅ 正确：保存前确保所有 NOT NULL 字段都有值
- 💡 提示：使用 `auth.uid()` 获取当前登录用户的 ID

#### 2. 前端开发相关

**数据加载问题：**
- ❌ 错误：只保存数据，忘记实现加载逻辑
- ✅ 正确：保存和加载功能要同时实现
- 💡 提示：编辑模式下，`onMounted` 时要加载数据

**数据结构不匹配：**
- ❌ 错误：保存时用新表结构，加载时还用旧表结构
- ✅ 正确：保存和加载要使用相同的表结构
- 💡 提示：修改数据库结构后，前端代码也要同步修改

**关联查询问题：**
- ❌ 错误：只查询主表，忘记查询关联表
- ✅ 正确：使用 `Promise.all` 并行查询关联数据
- 💡 示例：
  ```javascript
  // 查询课次
  const { data: sessions } = await supabase
    .from('training_sessions')
    .select('*')
    .eq('template_id', templateId)

  // 为每个课次查询动作
  const sessionsWithExercises = await Promise.all(
    sessions.map(async (session) => {
      const { data: exercises } = await supabase
        .from('session_exercises')
        .select('*')
        .eq('session_id', session.id)
      return { ...session, exercises }
    })
  )
  ```

**索引问题：**
- ❌ 错误：传递错误的索引值（如 `dayIndex` 为 null）
- ✅ 正确：确保索引值正确传递，并在使用前验证
- 💡 提示：添加调试日志 `console.log('dayIndex:', dayIndex)`
