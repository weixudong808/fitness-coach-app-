# Claude 训练记录助手 - 指令手册

## 🎯 功能说明

通过对话方式实时记录会员训练数据，直接写入数据库。

---

## 📋 命令格式

### 1. 开启训练记录

**触发词：**
- `开启 [会员名] 训练记录`
- `开始记录 [会员名]`
- `记录 [会员名] 训练`

**示例：**
```
开启张三训练记录
开始记录李四
记录王五训练
```

**执行流程：**
1. 查询会员信息（从 `members` 表）
2. 查询会员的活跃训练计划（从 `member_plans` 表，status='active'）
3. 查询计划下的待训练课次（从 `training_sessions` 表，completed=false）
4. 如果有多个待训练课次，选择 session_number 最小的
5. 显示会员信息和当前课次信息
6. 进入记录模式

**如果找不到会员：**
- 提示：找不到会员"XXX"，请确认会员姓名

**如果没有活跃计划：**
- 提示：会员"XXX"当前没有活跃的训练计划

**如果没有待训练课次：**
- 提示：会员"XXX"的所有课次都已完成

---

### 2. 记录训练动作

**输入格式（灵活识别）：**
```
[动作名称] [重量] [次数] [组数] [备注]
```

**支持的格式：**
- `卧推 80kg 10次 3组`
- `卧推 80 10 3`
- `卧推 80/10/3`
- `深蹲 100kg 8次 4组 感觉不错`
- `硬拉 120 6 3 下次加重`

**字段映射：**
- 动作名称 → `exercise_name`
- 重量 → `weight`（自动添加单位 kg）
- 次数 → `reps_standard`（格式：`10次`）
- 组数 → `sets`（数字）
- 备注 → `coach_comment`（可选）

**执行流程：**
1. 解析输入的动作数据
2. 立即写入 `session_exercises` 表
3. 关联到当前的 `session_id`
4. `order_index` 自动递增（查询当前课次已有的动作数量+1）
5. 返回确认信息

**数据库插入：**
```javascript
await supabase
  .from('session_exercises')
  .insert({
    session_id: '当前课次ID',
    exercise_name: '卧推',
    weight: '80kg',
    reps_standard: '10次',
    sets: 3,
    coach_comment: '感觉不错',
    order_index: 1
  })
```

---

### 3. 结束训练记录

**触发词：**
- `结束`
- `结束记录`
- `保存`
- `完成`

**执行流程：**
1. 更新 `training_sessions` 表：
   - `completed = true`
   - `completed_date = 今天的日期`
2. 统计本次训练的动作数量
3. 显示训练总结
4. 退出记录模式

**数据库更新：**
```javascript
await supabase
  .from('training_sessions')
  .update({
    completed: true,
    completed_date: new Date().toISOString().split('T')[0]
  })
  .eq('id', '当前课次ID')
```

---

### 4. 取消记录

**触发词：**
- `取消`
- `取消记录`
- `退出`

**执行流程：**
1. 不修改课次状态
2. 已记录的动作保留在数据库中
3. 退出记录模式

---

## 🔧 数据解析规则

### 重量解析
- 输入：`80kg`, `80`, `80.5kg`, `80.5`
- 输出：`80kg`, `80kg`, `80.5kg`, `80.5kg`
- 规则：如果没有单位，自动添加 `kg`

### 次数解析
- 输入：`10次`, `10`, `8-10`, `10-12次`
- 输出：`10次`, `10次`, `8-10次`, `10-12次`
- 规则：如果没有"次"，自动添加

### 组数解析
- 输入：`3`, `3组`, `4`
- 输出：`3`, `3`, `4`
- 规则：存储为数字，不带单位

### 备注解析
- 输入：动作数据后的所有文字
- 输出：原样存储到 `coach_comment`

## 📊 数据库表结构

### members（会员表）
- `id` - UUID
- `name` - 会员姓名

### member_plans（训练计划表）
- `id` - UUID
- `member_id` - 会员ID
- `template_id` - 模板ID
- `status` - 状态（active/completed/cancelled）

### training_sessions（训练课次表）
- `id` - UUID
- `template_id` - 模板ID
- `session_number` - 课次编号
- `core_focus` - 核心重点
- `training_part` - 训练部位
- `completed` - 是否完成（boolean）
- `completed_date` - 完成日期

### session_exercises（课次动作表）
- `id` - UUID
- `session_id` - 课次ID
- `exercise_name` - 动作名称
- `weight` - 重量
- `reps_standard` - 次数标准
- `sets` - 组数
- `coach_comment` - 教练评语
- `order_index` - 排序索引

## ⚠️ 注意事项

1. **数据立即写入**
   - 每条动作数据立即写入数据库
   - 不暂存，不等待"保存"命令

2. **课次状态更新**
   - 只有说"结束"/"完成"时，才更新课次为"已完成"
   - 如果说"取消"，课次保持"待训练"状态

3. **错误处理**
   - 如果解析失败，提示用户重新输入
   - 如果数据库写入失败，提示错误信息

4. **灵活性**
   - 支持多种输入格式
   - 备注是可选的
   - 可以随时取消

---

## 🔄 状态管理

**记录模式状态：**
- 当前会员ID
- 当前训练计划ID
- 当前课次ID
- 已记录的动作数量

**退出记录模式的条件：**
- 用户说"结束"/"完成"/"保存"
- 用户说"取消"/"退出"
- 用户开启另一个会员的训练记录
