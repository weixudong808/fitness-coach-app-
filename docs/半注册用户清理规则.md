# 半注册用户清理规则

## 问题说明
当用户注册时，如果 Supabase Auth 创建成功但业务表（coaches/members）插入失败，会产生"半注册用户"：
- Auth 表里有该用户
- 业务表里没有对应记录

这些"半注册用户"需要定期清理，避免脏数据积累。

## 清理规则

### 清理条件（必须同时满足）
1. Auth 表里存在该用户
2. 业务表（coaches 或 members）里没有对应的 user_id 记录
3. 用户创建时间超过 10 分钟（避免误删正在注册中的用户）
4. **只清理业务用户**：`user_metadata.user_type` 为 `'coach'` 或 `'member'`
5. **排除管理员账号**（满足以下任一条件的不清理）：
   - `user.user_metadata?.user_type === 'admin'`
   - `user.user_metadata?.role === 'admin'`
   - `user.app_metadata?.role === 'admin'`

### 执行频率
每天执行一次（建议在凌晨低峰期）

### 实现方式
**方案1：Edge Function（推荐）**
- 创建 `cleanup-orphaned-auth-users` Edge Function
- 使用 service_role 权限查询 Auth 表和业务表
- 删除符合条件的孤立 Auth 用户
- **排除 admin 角色用户**
- 记录清理日志

**方案2：定时脚本**
- 使用 cron job 或云函数定时执行
- 调用 Supabase Admin API 清理

## 清理逻辑伪代码（完整版）
```javascript
import { createClient } from '@supabase/supabase-js'

// 使用 service_role 权限
const supabaseAdmin = createClient(
  Deno.env.get('SUPABASE_URL'),
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')
)

// 获取所有 Auth 用户（需要分页处理）
let page = 1  // 从第1页开始
const perPage = 1000
let hasMore = true
let totalCleaned = 0

while (hasMore) {
  const { data: authData, error } = await supabaseAdmin.auth.admin.listUsers({
    page: page,
    perPage: perPage
  })

  if (error || !authData || authData.users.length === 0) {
    hasMore = false
    break
  }

  // 过滤：只处理业务用户 + 排除管理员 + 创建时间超过10分钟
  const candidateUsers = authData.users.filter(u => {
    // 只处理业务用户（coach 或 member）
    const userType = u.user_metadata?.user_type
    if (userType !== 'coach' && userType !== 'member') {
      return false
    }

    // 排除管理员
    const isAdmin =
      u.user_metadata?.user_type === 'admin' ||
      u.user_metadata?.role === 'admin' ||
      u.app_metadata?.role === 'admin'

    if (isAdmin) return false

    // 只处理创建超过10分钟的用户
    const createdAt = new Date(u.created_at)
    const tenMinutesAgo = Date.now() - 10 * 60 * 1000
    return createdAt < tenMinutesAgo
  })

  // 检查每个候选用户是否在业务表中
  for (const user of candidateUsers) {
    const userType = user.user_metadata?.user_type

    let existsInBusinessTable = false

    if (userType === 'coach') {
      const { data: coach } = await supabaseAdmin
        .from('coaches')
        .select('id')
        .eq('user_id', user.id)
        .single()

      existsInBusinessTable = !!coach
    } else if (userType === 'member') {
      const { data: member } = await supabaseAdmin
        .from('members')
        .select('id')
        .eq('user_id', user.id)
        .single()

      existsInBusinessTable = !!member
    }

    // 如果业务表中不存在，则删除该 Auth 用户
    if (!existsInBusinessTable) {
      const { error: deleteError } = await supabaseAdmin.auth.admin.deleteUser(user.id)

      if (!deleteError) {
        console.log(`✓ 清理孤立用户: ${user.id} (${userType}, phone: ${user.phone})`)
        totalCleaned++
      } else {
        console.error(`✗ 清理失败: ${user.id}, 错误: ${deleteError.message}`)
      }
    }
  }

  // 如果返回的用户数少于 perPage，说明已经是最后一页
  if (authData.users.length < perPage) {
    hasMore = false
  } else {
    page += 1
  }
}

console.log(`清理完成，共清理 ${totalCleaned} 个孤立用户`)
```
